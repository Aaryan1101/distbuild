distbuild
Distribution, Packaging & Publishing Guide

How to turn your build tool into something anyone can install in one command


GitHub Open Source  |  curl one-liner  |  apt + winget + scoop  |  Permission-based dependency installer


1. The Big Picture


Right now distbuild is a JAR on your machine. The goal of this guide is to get it to the point where any developer anywhere can install it with one command and start using it — the same way people install Rust, NVM, or the GitHub CLI.

Here is the complete distribution stack you will build:

LayerWhat it isWho uses itGitHub ReleasesVersioned .jar + installer scripts attached to each Git tagEverything else depends on thisinstall.shcurl | bash one-liner for LinuxLinux developers, CI serversinstall.ps1irm | iex one-liner for Windows PowerShellWindows developersapt packageDebian/Ubuntu package hosted on GitHub Pages as a custom APT repoLinux users who prefer apt installScoop manifestJSON manifest in a Scoop bucket for WindowsWindows users who prefer scoop installwinget manifestYAML manifest submitted to winget-pkgs repoWindows users on wingetDependency checkerBuilt into every installer — detects Java, Redis, asks to installEvery new user automatically
  You do NOT need all of these on day one. The recommended order is: GitHub Releases ? install.sh/ps1 ? Scoop ? apt ? winget. Each step adds more reach.

3. GitHub Actions Release Pipeline


Every time you push a git tag like v1.2.0, GitHub Actions automatically builds the fat JAR, attaches it to a GitHub Release, and triggers package manifest updates. You never manually upload anything.

3.1  .github/workflows/release.yml

name: Release
 
on:
  push:
    tags: ['v*']          # triggers on v1.0.0, v1.2.3, etc.
 
jobs:
  build-and-release:
    runs-on: ubuntu-latest
    permissions:
      contents: write     # needed to create GitHub Release
 
    steps:
      - uses: actions/checkout@v4
 
      - uses: actions/setup-java@v4
        with:
          java-version: '17'
          distribution: 'temurin'
 
      # Build the fat CLI JAR
      - name: Build CLI JAR
        run: ./gradlew :cli:jar
 
      # Rename JAR to include version from tag
      - name: Rename artifact
        run: |
          VERSION=${GITHUB_REF#refs/tags/v}
          cp cli/build/libs/distbuild.jar distbuild-${VERSION}.jar
          echo "VERSION=$VERSION" >> $GITHUB_ENV
 
      # Create GitHub Release and upload JAR + installers
      - name: Create Release
        uses: softprops/action-gh-release@v2
        with:
          files: |
            distbuild-${{ env.VERSION }}.jar
            installer/install.sh
            installer/install.ps1
            installer/uninstall.sh
            installer/uninstall.ps1
          generate_release_notes: true
 
      # Update Scoop manifest with new version + SHA256
      - name: Update Scoop manifest
        run: bash packages/scoop/update-manifest.sh ${{ env.VERSION }}
 
      # Rebuild apt repo on GitHub Pages
      - name: Build and publish apt package
        run: bash packages/apt/build-deb.sh ${{ env.VERSION }}

  To publish a new release: git tag v1.1.0 && git push origin v1.1.0 — that's it. The pipeline handles everything else.



4. Dependency Detection & Permission-Based Installer


This is the most important part. Before installing distbuild, the installer checks for Java 17+ and Redis. If either is missing, it clearly explains what is needed and asks the user's permission before installing anything.

The flow looks like this for every new user:

  Checking requirements for distbuild...
 
  [OK]  Java 17.0.10 (Temurin) — found at /usr/lib/jvm/java-17-openjdk
  [!!]  Redis — NOT FOUND
 
  distbuild requires Redis for the shared build cache.
  Without it the tool will still work, but build caching
  will be disabled and distributed builds will be slower.
 
  Install Redis now? [Y/n]: _

4.1  deps/check-deps.sh  (Linux)

#!/bin/bash
# Dependency checker — sourced by install.sh
# Returns 0 if all deps satisfied, 1 if something was declined
 
JAVA_MIN_VERSION=17
RED='\033[0;31m'  ; GREEN='\033[0;32m'
YELLOW='\033[1;33m'; RESET='\033[0m'
 
ok()   { echo -e "  ${GREEN}[OK]${RESET}  $1"; }
warn() { echo -e "  ${YELLOW}[!!]${RESET}  $1"; }
fail() { echo -e "  ${RED}[XX]${RESET}  $1"; }
 
ask_permission() {
  local prompt="$1"
  read -rp "  $prompt [Y/n]: " ans
  [[ "${ans:-Y}" =~ ^[Yy] ]]
}
 
# ?? Check Java ???????????????????????????????????????????????????????
check_java() {
  if command -v java &>/dev/null; then
    VER=$(java -version 2>&1 | awk -F'"' '/version/{print $2}' | cut -d. -f1)
    if [ "$VER" -ge "$JAVA_MIN_VERSION" ] 2>/dev/null; then
      ok "Java $VER detected"
      return 0
    else
      warn "Java $VER detected — distbuild needs Java $JAVA_MIN_VERSION+"
    fi
  else
    warn "Java — NOT FOUND"
  fi
 
  echo ""
  echo "  distbuild requires Java $JAVA_MIN_VERSION or higher."
  echo "  This installer will use SDKMAN to install Temurin JDK $JAVA_MIN_VERSION."
  echo ""
 
  if ask_permission "Install Java $JAVA_MIN_VERSION (Temurin) now?"; then
    if ! command -v sdk &>/dev/null; then
      echo "  Installing SDKMAN first..."
      curl -s https://get.sdkman.io | bash
      source "$HOME/.sdkman/bin/sdkman-init.sh"
    fi
    sdk install java 17.0.10-tem
    ok "Java 17 installed via SDKMAN"
    return 0
  else
    echo "  Skipped. You can install Java manually: https://adoptium.net"
    echo "  distbuild will not work without Java $JAVA_MIN_VERSION+."
    return 1
  fi
}
 
# ?? Check Redis ??????????????????????????????????????????????????????
check_redis() {
  if command -v redis-server &>/dev/null; then
    VER=$(redis-server --version | awk '{print $3}' | cut -dv -f2)
    ok "Redis $VER detected"
    return 0
  fi
 
  warn "Redis — NOT FOUND"
  echo ""
  echo "  Redis is used by distbuild as the shared build cache."
  echo "  Without it: caching disabled, builds will be slower."
  echo "  It only needs to run on ONE machine (the coordinator's)."
  echo ""
 
  if ask_permission "Install Redis now?"; then
    if command -v apt-get &>/dev/null; then
      sudo apt-get install -y redis-server
      sudo systemctl enable --now redis-server
    elif command -v dnf &>/dev/null; then
      sudo dnf install -y redis
      sudo systemctl enable --now redis
    else
      echo "  Cannot detect package manager. Install Redis manually:"
      echo "  https://redis.io/docs/getting-started/installation/"
      return 1
    fi
    ok "Redis installed and started"
    return 0
  else
    echo "  Skipped. You can install Redis later from: https://redis.io"
    echo "  Note: build caching will be disabled until Redis is available."
    return 0   # non-fatal — tool still works, just slower
  fi
}
 
# ?? Run all checks ???????????????????????????????????????????????????
run_checks() {
  echo ""
  echo "  Checking requirements for distbuild..."
  echo ""
  check_java || return 1
  check_redis
  echo ""
}

4.2  deps/check-deps.ps1  (Windows)

# Dependency checker for Windows PowerShell
 
function Write-Ok   { param($msg) Write-Host "  [OK]  $msg" -ForegroundColor Green  }
function Write-Warn { param($msg) Write-Host "  [!!]  $msg" -ForegroundColor Yellow }
function Write-Fail { param($msg) Write-Host "  [XX]  $msg" -ForegroundColor Red    }
 
function Ask-Permission {
  param([string]$Prompt)
  $ans = Read-Host "  $Prompt [Y/n]"
  return ($ans -eq '' -or $ans -match '^[Yy]')
}
 
function Check-Java {
  $java = Get-Command java -ErrorAction SilentlyContinue
  if ($java) {
    $ver = (java -version 2>&1)[0] -replace '.*version "(\d+).*', '$1'
    if ([int]$ver -ge 17) {
      Write-Ok "Java $ver detected"
      return $true
    }
    Write-Warn "Java $ver found — distbuild needs Java 17+"
  } else {
    Write-Warn "Java — NOT FOUND"
  }
 
  Write-Host ""
  Write-Host "  distbuild requires Java 17 or higher."
  Write-Host "  This installer will use winget to install Microsoft OpenJDK 17."
  Write-Host ""
 
  if (Ask-Permission 'Install Java 17 (Microsoft OpenJDK) now?') {
    winget install Microsoft.OpenJDK.17 --silent --accept-package-agreements
    Write-Ok 'Java 17 installed'
    return $true
  } else {
    Write-Host '  Skipped. Download manually: https://adoptium.net'
    Write-Fail 'distbuild will not work without Java 17+'
    return $false
  }
}
 
function Check-Redis {
  $redis = Get-Command redis-server -ErrorAction SilentlyContinue
  if ($redis) { Write-Ok 'Redis detected'; return $true }
 
  Write-Warn 'Redis — NOT FOUND'
  Write-Host ''
  Write-Host '  Redis is used for the shared build cache.'
  Write-Host '  Without it: caching disabled, builds will be slower.'
  Write-Host '  It only needs to run on the coordinator machine.'
  Write-Host ''
 
  if (Ask-Permission 'Install Redis (via Scoop) now?') {
    if (-not (Get-Command scoop -ErrorAction SilentlyContinue)) {
      Write-Host '  Installing Scoop first...'
      irm get.scoop.sh | iex
    }
    scoop install redis
    Write-Ok 'Redis installed via Scoop'
    return $true
  } else {
    Write-Host '  Skipped. Install later: https://redis.io/docs/getting-started'
    Write-Host '  Build caching will be disabled until Redis is available.'
    return $true   # non-fatal
  }
}
 
function Run-Checks {
  Write-Host ''
  Write-Host '  Checking requirements for distbuild...'
  Write-Host ''
  if (-not (Check-Java))  { return $false }
  Check-Redis | Out-Null
  Write-Host ''
  return $true
}



5. The One-Liner Installers


These are the two commands you will publish in your README. They download the latest release from GitHub, run dependency checks, install the JAR, and add distbuild to PATH.

5.1  install.sh  —  Linux one-liner

The command users will run:

$ curl -fsSL https://raw.githubusercontent.com/YOUR_USERNAME/distbuild/main/installer/install.sh | bash

Full install.sh content:

#!/bin/bash
set -euo pipefail
 
REPO="YOUR_USERNAME/distbuild"
INSTALL_DIR="/opt/distbuild"
BIN_DIR="/usr/local/bin"
CONFIG_DIR="$HOME/.distbuild"
 
# ?? Fetch latest version from GitHub API ?????????????????????????????

# ?? Run dependency checks (Java, Redis) ??????????????????????????????

# ?? Download the JAR ?????????????????????????????????????????????????

LAUNCHER

  defaultType: debug

fi

$ErrorActionPreference = 'Stop'

New-Item -ItemType Directory -Force -Path $InstallDir | Out-Null
New-Item -ItemType Directory -Force -Path $BinDir     | Out-Null

@'
@echo off
java %DISTBUILD_JVM_OPTS% -jar C:\distbuild\distbuild.jar %*
'@ | Set-Content "$BinDir\distbuild.bat"
 
# ?? Add to PATH ???????????????????????????????????????????????????????
$curPath = [Environment]::GetEnvironmentVariable('Path', 'User')
if ($curPath -notlike "*$BinDir*") {
  [Environment]::SetEnvironmentVariable('Path', "$curPath;$BinDir", 'User')
  Write-Host '  Added to PATH (open a new terminal to use)'
}
 
# ?? Write default config ?????????????????????????????????????????????

$configFile = "$ConfigDir\config.yml"

6. Package Managers


Package managers let users install distbuild the same way they install any other tool on their system. Each one is a separate small file to maintain.

6.1  Scoop  (Windows — recommended first)

Scoop is the most popular Windows package manager for developer tools. Setting up a custom Scoop 'bucket' takes about 20 minutes and means Windows users can install with:

> scoop bucket add distbuild https://github.com/YOUR_USERNAME/scoop-distbuild
> scoop install distbuild

Step 1 — Create a separate bucket repository
Scoop buckets are GitHub repos. Create a new repo named scoop-distbuild (separate from your main project).

Step 2 — Create the manifest  (scoop-distbuild/distbuild.json)

{
  "version": "1.0.0",
  "description": "Distributed Java build system",
  "homepage": "https://github.com/YOUR_USERNAME/distbuild",
  "license": "Apache-2.0",
  "architecture": {
    "64bit": {
      "url": "https://github.com/YOUR_USERNAME/distbuild/releases/download/v1.0.0/distbuild-1.0.0.jar",
      "hash": "sha256:PASTE_SHA256_HERE"
    }
  },
  "pre_install": [
    "if (!(Get-Command java -ErrorAction SilentlyContinue)) {",
    "  Write-Warning 'Java 17+ not found. Run: winget install Microsoft.OpenJDK.17'",
    "}"
  ],
  "bin": [["distbuild.jar", "distbuild", "java -jar '$dir\\distbuild.jar' $args"]],
  "checkver": {
    "github": "https://github.com/YOUR_USERNAME/distbuild"
  },
  "autoupdate": {
    "architecture": {
      "64bit": {
        "url": "https://github.com/YOUR_USERNAME/distbuild/releases/download/v$version/distbuild-$version.jar"
      }
    }
  }
}

Step 3 — Auto-update on release (packages/scoop/update-manifest.sh)

#!/bin/bash
# Called by release.yml after a new tag is pushed
VERSION=$1
JAR_URL="https://github.com/YOUR_USERNAME/distbuild/releases/download/v${VERSION}/distbuild-${VERSION}.jar"
 
# Compute SHA256 of the released JAR
SHA=$(curl -fsSL $JAR_URL | sha256sum | awk '{print $1}')
 
# Clone the scoop bucket, update the manifest, push
git clone https://github.com/YOUR_USERNAME/scoop-distbuild.git /tmp/scoop-bucket
cd /tmp/scoop-bucket
jq --arg v "$VERSION" --arg h "sha256:$SHA" \
   --arg u "$JAR_URL" \
   '.version=$v | .architecture["64bit"].url=$u | .architecture["64bit"].hash=$h' \
   distbuild.json > tmp.json && mv tmp.json distbuild.json
git add distbuild.json
git commit -m "Update distbuild to v${VERSION}"
git push

6.2  apt  (Debian / Ubuntu)

You host a custom APT repository on GitHub Pages. Once set up, Linux users install with:

$ curl -fsSL https://YOUR_USERNAME.github.io/distbuild/gpg.key | sudo gpg --dearmor -o /etc/apt/keyrings/distbuild.gpg
$ echo 'deb [signed-by=/etc/apt/keyrings/distbuild.gpg] https://YOUR_USERNAME.github.io/distbuild stable main' \
    | sudo tee /etc/apt/sources.list.d/distbuild.list
$ sudo apt update && sudo apt install distbuild

packages/apt/distbuild.control  (Debian package metadata)

Package: distbuild
Version: 1.0.0
Section: devel
Priority: optional
Architecture: all
Depends: default-jre-headless (>= 2:1.17)
Recommends: redis-server
Maintainer: Your Name <you@example.com>
Homepage: https://github.com/YOUR_USERNAME/distbuild
Description: Distributed Java build system
 Speeds up Gradle builds by distributing compilation
 tasks across multiple machines on your local network.

  The Depends: field means apt automatically installs Java if missing. Redis is Recommends — apt will suggest it but not require it.

packages/apt/build-deb.sh  (called by release.yml)

#!/bin/bash
# Builds a .deb and updates the GitHub Pages apt repo
set -euo pipefail
VERSION=$1
 
# Build .deb package structure
mkdir -p deb-pkg/DEBIAN
mkdir -p deb-pkg/opt/distbuild
mkdir -p deb-pkg/usr/local/bin
 
# Copy control file, substitute version
sed "s/Version: .*/Version: $VERSION/" packages/apt/distbuild.control > deb-pkg/DEBIAN/control
 
# Copy JAR
cp distbuild-${VERSION}.jar deb-pkg/opt/distbuild/distbuild.jar
 
# Write launcher
cat > deb-pkg/usr/local/bin/distbuild <<'EOF'
#!/bin/bash
exec java ${DISTBUILD_JVM_OPTS:--Xmx512m} -jar /opt/distbuild/distbuild.jar "$@"
EOF
chmod +x deb-pkg/usr/local/bin/distbuild
 
# Build the .deb
dpkg-deb --build deb-pkg distbuild-${VERSION}.deb
 
# Update GitHub Pages apt repository (requires gh-pages branch)
git clone --branch gh-pages https://github.com/YOUR_USERNAME/distbuild.git apt-repo
cp distbuild-${VERSION}.deb apt-repo/
cd apt-repo
dpkg-scanpackages --multiversion . > Packages
gzip -k -f Packages
apt-ftparchive release . > Release
gpg --default-key you@example.com -abs -o Release.gpg Release
gpg --default-key you@example.com --clearsign -o InRelease Release
git add -A && git commit -m "apt repo: distbuild v${VERSION}"
git push

6.3  winget  (Windows — submit to Microsoft's registry)

winget is built into Windows 11. To get listed you submit a PR to github.com/microsoft/winget-pkgs. This is the longest process (takes 1–3 days for review) but gives the most reach.

packages/winget/distbuild.yaml

PackageIdentifier: YourName.distbuild
PackageVersion: 1.0.0
PackageLocale: en-US
Publisher: Your Name
PackageName: distbuild
License: Apache-2.0
ShortDescription: Distributed Java build system
Description: Speeds up Gradle builds by distributing compilation across your team's machines.
Tags:
  - java
  - build
  - gradle
  - developer-tools
Installers:
  - Architecture: x64
    InstallerType: script
    InstallerUrl: https://raw.githubusercontent.com/YOUR_USERNAME/distbuild/main/installer/install.ps1
    InstallerSha256: PASTE_SHA256_HERE
    InstallerSwitches:
      Silent: '-NonInteractive'
ManifestType: singleton
ManifestVersion: 1.4.0

  After your first release: fork winget-pkgs, add your manifest under manifests/y/YourName/distbuild/1.0.0/, and submit a PR. Microsoft's bot validates and merges it within 1–3 days.



7. Version Management


Follow semantic versioning: MAJOR.MINOR.PATCH (e.g. 1.2.3). The version lives in one place — gradle.properties — and everything else reads it from there.

# gradle.properties
distbuildVersion=1.0.0
 
# The CLI JAR embeds this at build time:
# DistBuildCli.java ? @Command(version = "distbuild " + BuildInfo.VERSION)

9. Rollout Order (Recommended)


Don't try to do everything at once. Here is the recommended order — each step gives you a working distribution channel before moving to the next.

WeekWhat to buildUnlocksWeek 1GitHub Actions release.yml, install.sh, install.ps1, dependency checkerOne-liner install works on Linux + WindowsWeek 2Scoop bucket (scoop-distbuild repo + manifest)Windows: scoop install distbuildWeek 3apt repo on GitHub Pages + build-deb.shLinux: apt install distbuildWeek 4winget manifest PR to microsoft/winget-pkgsWindows 11: winget install distbuildOngoingAuto-update scripts in release.ymlAll channels update automatically on every release
  Start with the one-liner. Share it with your team. Get feedback. Then invest in package managers once the tool is stable.
